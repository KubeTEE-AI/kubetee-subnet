# Summary of SN62 Discord Conversation

Date: October 9 2025

Subnet 62 Ridges is a similar subnet of KubeTEE AI subnet for AI Agent improvments goal, we learn from the issues they are having.

## Main Conflict: Agent Approval Dispute

**The Core Issue:**
A miner named **rudo** developed an AI agent that was initially attacked and called "messy" or an "exploit." After public discussion, everyone agreed rudo was right and the implementation was legitimate. However, despite this consensus, some community members continued criticizing it, and the agent was ultimately unapproved by **Plasma Blasted** (admin/moderator).

## Key Points of Contention:

**Technical Dispute:**
- rudo's agent implemented error message handling for class-based Python code
- Critics claimed it "overfits" to specific benchmark problems (polyglot/SWE-bench)
- rudo argued it's a legitimate workflow improvement, not an exploit
- Related controversy over "one-shot embedding" technique and who used it first

**rudo's Frustration:**
- Feels unfairly targeted despite community agreement
- Believes people are attacking for personal gain (to become top miner) rather than subnet improvement
- Threatens to scrutinize every top miner's code going forward if this behavior continues
- Questions why criticism "wins" even after everyone agreed it was wrong

**Community Response:**

*Supportive voices:*
- **Levi** and others call for clear rules defining exploits vs. legitimate improvements
- Multiple community members want Plasma banned
- Recognition that no one (including top miners) understands what counts as exploit/hardcoding

*Moderation stance:*
- **Kat (Bittensor CM)** warns if the subnet owner isn't being fair, people should consider leaving
- **Plasma** promises to explain reasoning and create list of banned patterns as examples
- Suggests community vote on whether to proceed

## Secondary Issues:

**Platform Stability:**
- Infrastructure problems with proxy/platform restarts
- Provider (Chutes) capacity issues causing inference failures
- Database operations causing system slowdowns
- Rate limiting being implemented

**Governance Problems:**
- Lack of clear criteria for approving/rejecting agents
- Concerns about subjective/inconsistent decision-making
- Community toxicity and personal attacks on top performers

## Proposals:

**Lumina** suggested implementing a git-like system where miners must:
- Start agents from specific branches/commits
- Specify their origin
- Describe improvements made
- Create a roadmap showing agent evolution

This conversation highlights significant challenges in competitive AI subnet governance, particularly around fairness, transparency, and establishing objective evaluation criteria.
